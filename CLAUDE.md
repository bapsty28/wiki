# Wiki.js — Project Overview

## What this is

A self-hosted [Wiki.js 2.x](https://js.wiki) instance forked from [requarks/wiki](https://github.com/requarks/wiki) for customization and self-hosting on a Hetzner VPS.

Live site: https://wiki.joshuabapst.com

## Architecture

```
GitHub (bapsty28/wiki)
    │
    │  git push to main
    ▼
GitHub Actions (CI/CD)
    │  1. yarn install
    │  2. yarn build (webpack)
    │  3. rsync built files to VPS
    │  4. yarn install --production on VPS
    │  5. systemctl restart wikijs
    ▼
Hetzner VPS (CPX11, Ubuntu 24.04) — 5.78.227.55
    │
    ├── Caddy (reverse proxy + automatic HTTPS via Let's Encrypt)
    │       wiki.joshuabapst.com → localhost:3000
    │
    ├── Wiki.js (systemd service: wikijs)
    │       /opt/wikijs — production files
    │       runs as: wikijs (system user)
    │       Node.js 20 LTS
    │
    └── PostgreSQL
            database: wiki
            user: wikijs
```

## Local development

**Prerequisites:** Node.js 24, Yarn 1.x

**First time setup:**
```powershell
yarn install
```

**Start dev server** (webpack hot-reload + auto server restart):
```powershell
yarn dev
```

Open http://localhost:3000

The local instance uses **SQLite** (stored at `data/db.sqlite`) — no database setup required.
`config.yml` and `data/` are gitignored and never committed.

**Stopping the dev server:** Ctrl+C in the terminal running `yarn dev`.

## Deployment

Push to `main` — GitHub Actions handles the rest. The workflow is at `.github/workflows/deploy.yml`.

**What gets deployed:**
- Built frontend assets (`assets/`, `server/views/*.pug` generated by webpack)
- Server-side code (`server/`)
- `package.json`

**What is NOT deployed** (lives only on the VPS):
- `config.yml` (production database credentials)
- `data/` (uploads, cache)
- `node_modules/` (reinstalled on VPS from package.json)

## VPS access

```bash
ssh root@5.78.227.55
```

**Useful commands on VPS:**
```bash
systemctl status wikijs          # check service status
journalctl -u wikijs -f          # tail logs
systemctl restart wikijs         # restart after manual changes
```

## Customization approach

This is a fork of Wiki.js 2.x. Customizations live in this repo and are deployed via CI/CD.

**Client-side** (Vue.js components, SCSS): `client/`
**Server-side** (GraphQL, auth, storage): `server/`
**Webpack config**: `dev/webpack/`

After client-side changes, `yarn build` must run to produce updated assets — this happens automatically in CI.

After server-side only changes, just a service restart is needed (CI handles this too).

## Pulling upstream updates

To merge new releases from the original Wiki.js repo:

```bash
git remote add upstream https://github.com/requarks/wiki.git
git fetch upstream
git merge upstream/main
# resolve any conflicts
git push origin main
```
